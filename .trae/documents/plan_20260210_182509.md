Vou implementar um sistema de configurações fiscais anuais para armazenar e gerenciar a tabela do IRPF no banco de dados, permitindo ajustes flexíveis a cada ano.

### 1. Modelo de Dados e Tipagem
Atualizar `types/financial/financial.ts` para incluir as definições da tabela IRPF:
```typescript
export interface IrpfRange {
  limit: number;      // Limite superior da faixa (em centavos)
  rate: number;       // Alíquota (ex: 0.075 para 7.5%)
  deduction: number;  // Parcela a deduzir (em centavos)
}

export interface IrpfConfig {
  year: number;
  ranges: IrpfRange[];
  updatedAt?: any;
}
```

### 2. Backend (API)
Criar `app/api/admin/finance/fiscal-config/route.ts`:
- **GET**: Busca a configuração do ano especificado (`?year=2024`).
- **POST**: Salva/Atualiza a configuração para o ano.
- Armazenamento: Coleção `irpf_tables` no Firestore, com ID igual ao ano (ex: documento `2024`).

### 3. Frontend (Componente de Configuração)
Criar `components/financial/FiscalConfigModal.tsx`:
- Um modal (Dialog) acessível pelo dashboard.
- Formulário para editar os valores das 5 faixas do IRPF (Isento + 4 faixas).
- Campos monetários formatados (R$).
- Botão para salvar que chama a API.

### 4. Integração no Dashboard
Atualizar `app/[locale]/hub/admin/finances/fiscal/page.tsx`:
- Adicionar botão de "Configurações" (ícone de engrenagem) ao lado do seletor de ano.
- Buscar a configuração da API ao carregar o ano.
- Atualizar a função `calculateAnnualIRPF` para usar os valores dinâmicos do banco (com fallback para os valores de 2024 se não houver config salva).

Isso permitirá que você ajuste os valores da tabela IRPF diretamente pela interface sempre que houver mudanças na legislação, sem precisar alterar o código.
