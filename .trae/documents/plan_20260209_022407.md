Implementar a funcionalidade de "Replay" de sessões anteriores utilizando XP como moeda.

### 1. Backend (`actions/srsActions.ts`)
*   **Nova Action**: `purchaseReplaySession(planId: string, dayIndex: number)`
    *   Calcula o custo: `50 + (diferença_dias * 10)`.
    *   Verifica saldo de XP do usuário.
    *   Deduz o XP e registra a transação (atualiza `currentXP`).
*   **Atualização**: `getDailyPractice(planId: string, dayOverride?: number)`
    *   Aceitar um argumento opcional `dayOverride`.
    *   Se fornecido, gera a prática para aquele dia específico, ignorando o progresso atual.
*   **Atualização**: `processPracticeResults(..., isReplay?: boolean)`
    *   Aceitar flag `isReplay`.
    *   Se `true`, **pula** a atualização de SRS e a concessão de XP/Streak.

### 2. Frontend - Timeline (`components/notebook/LearningPath.tsx`)
*   Receber `userXP` e `planId` como props.
*   Identificar dias passados (status `completed`).
*   Torná-los clicáveis.
*   Ao clicar:
    *   Calcular custo estimado.
    *   Exibir Modal de Confirmação ("Replay Day X? Custo: Y XP").
    *   Ao confirmar, chamar `purchaseReplaySession`.
    *   Sucesso: Redirecionar para `/hub/student/my-practice?day=X&replay=true`.

### 3. Frontend - Integração (`my-notebook/page.tsx`)
*   Obter `plan.id` e `student.gamification.currentXP` através do hook `useStudentPanel`.
*   Passar esses dados para o `LearningPath`.

### 4. Frontend - Sessão de Prática (`my-practice/page.tsx` & `PracticeSession.tsx`)
*   Ler parâmetros de URL `day` e `replay`.
*   Passar para `getDailyPractice` para carregar o conteúdo correto.
*   Passar flag `isReplay` para `processPracticeResults` ao finalizar, garantindo que não haja ganho de XP extra.
*   Desativar salvamento de progresso (resume) para sessões de replay para evitar conflito com a sessão atual.
