Vou alterar o arquivo `components/students/StudentProfileList.tsx` para incluir usuários sem perfil e adicionar os filtros solicitados.

### 1. Busca de Usuários

* Adicionar um `useEffect` para carregar automaticamente os usuários com role `student` ao montar o componente (atualmente ele só busca ao criar um novo usuário).

### 2. Lógica de Filtragem e Combinação

Vou criar uma lista unificada combinando `StudentProfile` e `User`.
Os filtros funcionarão da seguinte forma:

* **@ (Todos)**: Exibe tudo (Perfis + Usuários sem perfil).

* **Perfil**: Exibe apenas perfis que **não** estão associados a uma conta de usuário.

* **Perfil + Conta**: Exibe perfis que **já estão** associados a uma conta.

* **Conta**: Exibe usuários do sistema que **ainda não** possuem perfil de estudante criado.

### 3. Interface (UI)

* **Novo Filtro**: Adicionar um componente `Select` ao lado da barra de busca com as opções acima.

* **Renderização Condicional**:

  * Se o item for um **Perfil**: Mantém o Card atual.

  * Se o item for um **Usuário**: Renderiza um novo Card simplificado mostrando Nome, Email e um botão **"Criar Perfil"**.

### 4. Ação "Criar Perfil"

* O botão "Criar Perfil" no card de usuário levará para a página de criação, passando os dados do usuário via URL (`?userId=...&name=...&email=...`) para facilitar o preenchimento (caso implementado na página de destino).
